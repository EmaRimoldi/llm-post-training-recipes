# Milestone 1: Preference Data Collection And Review

This milestone covers the early data-centric part of the course project.

## Scope

The core goals were:

1. collect preference-style supervision for a set of STEM questions;
2. structure the submission data correctly;
3. support a literature-review deliverable for the team report.

## What Is In This Folder

- `data/`: collected preference data and exploratory cleaning notebooks.
- `_tests/`: validation scripts used to verify the expected submission format.
- `_artifacts/`: annotation support material.
- `pdfs/`: report output location.

## Why It Matters In A Portfolio

This milestone is less about model training and more about data operations:

- manual annotation support;
- JSON validation and data hygiene;
- reproducible submission structure.

That work is still relevant in NLP engineering because weak data pipelines usually become weak training pipelines later.

## Notes

- Much of the original folder structure came from the course template.
- For a public portfolio, the most relevant pieces are the data workflow and validation scripts rather than the assignment scaffolding.
